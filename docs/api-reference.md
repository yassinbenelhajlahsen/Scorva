# Scorva ‚Äî API & Route Reference

## API endpoints (all under `/api`)
- `GET /:league/teams`
- `GET /:league/teams/:teamId/seasons` ‚Äî distinct seasons the team has played in, ordered DESC
- `GET /:league/teams/:teamId/roster` ‚Äî optional `?season=`; returns roster sorted by league-specific per-game stat (NBA: points, NFL: yds, NHL: g+a). Each player includes `id, name, position, jerseynum, image_url, status, status_description, status_updated_at, espn_playerid, averages`. Cached 5m current / 30d past (`roster:{league}:{teamId}:{season}`)
- `GET /:league/standings` ‚Äî optional `?season=`; returns array of team objects sorted by conference then primary value. Each team includes `wins`, `losses`, `division`, and league-specific fields:
  - **NBA**: `winPct`, `pointDiff`, `confWinPct`
  - **NHL**: `otl`, `ptsPct`, `regWins`, `gf`, `pointDiff`, `confWinPct`
  - **NFL**: `ties`, `winPct = (W + 0.5√óT) / GP`, `divWinPct`, `confWinPct`, `pointDiff`; standings record shows `W-L-T` when ties > 0
  - Tiebreaker cascade is league-specific:
    - **NBA**: H2H record ‚Üí division-leader bonus (leader of own division ranks above non-leader) ‚Üí conf% ‚Üí point diff ‚Üí id
    - **NFL**: H2H (3+-way only when every pair has played each other; W/L/T counted) ‚Üí divWinPct ‚Üí confWinPct ‚Üí point diff ‚Üí id
    - **NHL**: regWins ‚Üí H2H points (W=2, OTL=1, L=0) ‚Üí goal diff ‚Üí goals for ‚Üí id; OTL counted when `ot1 IS NOT NULL` OR `fourthqtr IS NOT NULL` (period 4/5 scores, independent of ESPN status string)
  - Cached 5m current / 30d past
- `GET /:league/playoffs` ‚Äî supports `nba`, `nhl`, and `nfl` (400 for others); optional `?season=`; cached `playoffs:{league}:{season}` 30s current / 30d historical
  - **NBA response**: `{ season, isProjected, playIn, bracket: { eastern, western, finals } }` ‚Äî `playIn` null for pre-play-in seasons
  - **NHL response**: `{ season, isProjected, playIn: null, bracket: { eastern, western, finals } }` ‚Äî no play-in; also returns `{ season, unsupported: true }` for pre-2013-14 and 2019-20 seasons, or `{ season, isProjected: true, warning: "division_data_missing", bracket: {‚Ä¶} }` when team division metadata is missing
  - **NFL response**: `{ season, isProjected, format: "14team" | "12team", bracket: { afc, nfc, superBowl } }` ‚Äî no play-in; single-elimination; each conf block is `{ wildCard, divisional, confChampionship }`; `wildCard` has 3 series (14-team / 2020-21+) or 2 series (12-team / 2015-16 to 2019-20); seasons before 2015-16 return `{ season, unsupported: true }`; missing division data returns `{ season, isProjected: true, format, warning: "division_data_missing", bracket: {‚Ä¶} }`
- `GET /:league/games` ‚Äî optional `?date=YYYY-MM-DD` returns `{ games[], resolvedDate, resolvedSeason }` instead of a flat array; nearest-date fallback when no games exist on the requested date; validates format, 400 on mismatch
- `GET /:league/games/dates` ‚Äî optional `?season=`; returns `[{ date: "YYYY-MM-DD", count: N }]` for all dates with games in the season (cached 5 min)
- `GET /:league/games/:gameId` ‚Äî game detail. Each player's stats object on the boxscore includes `rating` (raw open-ended sum, NBA only ‚Äî null for NFL/NHL or when not yet computed) and `ratingGrade` (calibrated 0.0‚Äì10.0). For NBA/NHL playoff games, the response includes `seriesScore: { homeWins, awayWins }` (counted up through this game) and `hasUnplayedPriorSeriesGames: boolean` ‚Äî true when any earlier game in the same series is not yet `Final`. The frontend uses this to hide the prediction card for future playoff games whose prior series game hasn't been played yet (predictions for those would be effectively static until the prior game finishes). Always `false` for NFL (single-elimination) and regular-season games.
- `GET /:league/games/:gameId/plays` ‚Äî returns `{ plays[], source }` where `source` is `"db"` (Final, cached 30d), `"espn"` (live, proxied), or `"none"` (scheduled/no eventid)
- `GET /:league/games/:gameId/prediction` ‚Äî pre-game only (returns `null` for live/final); response: `{ homeTeam, awayTeam, headToHead, keyFactors, confidence }`. Each team object includes ratings, records, recent form (last 5), and `injuries: { impactFactor, players[] }` where each player has `{ id, name, position, imageUrl, status, statusDescription, statusUpdatedAt, impactShare, availability }`. `keyFactors` types: `home | injury | h2h | form | offense | defense | record` (max 5). Confidence drops to `"low"` if either team has <5 games or `impactFactor > 0.25`. Cached 1h, key includes `MAX(players.status_updated_at)` so a new ESPN injury report invalidates. See [ARCHITECTURE.md ¬ß Game prediction](./ARCHITECTURE.md#game-prediction-servicesgamespredictionservicejs) for the full algorithm.
- `GET /:league/games/:eventId/win-probability?final=true|false` ‚Äî returns `{ winProbability[], scoreMargin[] }`; proxied from ESPN summary; cached 30s (live) / 30d (final, `cacheIf` non-null); key `winprob:{league}:{espnEventId}`
- `GET /:league/players`
- `GET /:league/players/:slug` ‚Äî `:slug` is `lower(name)` with punctuation stripped and whitespace collapsed to dashes; for shared names the lowest-id player owns the bare slug, others use `slug-{id}` (e.g. `chris-paul-3144`). `getPlayerIdBySlug` resolves both forms. Game log rows include `rating` + `ratingGrade` (NBA only; null elsewhere).
- `GET /:league/players/duplicate-slugs` ‚Äî returns `{ [slug]: canonicalId }` for slugs shared by 2+ players in the league; cached 24h. Used by the frontend to render `slug-{id}` links for non-canonical duplicates.
- `GET /:league/players/:slug/similar` ‚Äî returns `{ players: [...] }` (up to 5); position-filtered for NFL/NHL; requires embeddings to be computed; returns `{ players: [] }` if player has < 5 games (< 2 for NFL)
- `GET /:league/head-to-head` ‚Äî query params `type` (`players` or `teams`) and `ids` (comma-separated, exactly 2 numeric IDs); returns `{ games: [...] }` with up to 20 recent Final games where the entities faced each other; for players, finds games where both appeared in stats; cached 30d
- `GET /:league/seasons`
- `GET /:league/players/:slug/streak` ‚Äî returns `{ streak: { length, statLabel, subjectType: "player" } | null }` for the player's currently active stat streak (or null if none); cached 60s
- `GET /:league/teams/:teamId/streak` ‚Äî returns `{ streak: { length, statLabel: "win"|"loss", subjectType: "team" } | null }` for the team's current W/L streak; cached 60s
- `GET /reports` ‚Äî backs the Pulse page (frontend route `/pulse`). Query params `?league=nba|nfl|nhl` (optional; omit for all-leagues), `?type=injury|move|birthday|streak` (optional), `?limit=` (default 20, max 50), `?offset=` (default 0). Returns `{ reports[], total, hasMore }`. Streaks are sourced from the `streak_events` table (active rows in current season); birthdays filter to current-season-active players; moves are parsed from ESPN transactions via `movesParser.js`. Cached 30 min per `(league, type)` combo. Cache is warmed post-upsert and prefetched on `/pulse` nav-link hover.
- `GET /news` ‚Äî optional `?limit=` (default 4, max 10); returns `{ articles: [{ headline, description, url, imageUrl, published, league }] }`; merged from all 3 leagues sorted by date with at least 1 per league; cached 5 min; filters out roundup/tracker articles
- `GET /:league/top-performances` ‚Äî NBA only in v1 (returns 400 for nfl/nhl). Includes both Final and live (in-progress) games ‚Äî live ratings recompute each `liveSync` cycle and surface immediately. Query params:
  - `?type=rankings|performances|plays` (default: `performances`; aliases: `cumulative‚Üírankings`, `games‚Üíperformances`)
  - `?window=today|week|month|season|all` (default: `week`; legacy alias: `?days=N` maps to `‚â§1‚Üítoday`, `‚â§7‚Üíweek`, else `month`)
  - `?sort=desc|asc` (default: `desc`; applies to ORDER BY direction for all three types)
  - `?position=all|G|F|C` (default: `all`; filters by player position via case-insensitive prefix match)
  - `?limit=N` (capped at 25)
  - `type=rankings`: per-player cumulative sum over window ‚Äî each entry has `{ player, totalRating, gamesPlayed, avgPerGame, bestGame }`; click target is `/:league/players/:slug`. `type=performances`: best/worst single-game performances ‚Äî each entry has `{ player, game, rating, ratingGrade, stats: { points, rebounds, assists } }`; click target is `/:league/games/:gameId`. `type=plays`: best/worst single plays ‚Äî each entry has `{ player, game, play }`. When `type=plays`, each row includes a `play` object: `{id, description, period, clock, weightedValue, wpaDelta}`. For `performances` and `plays`, the `game` object includes `isLive` (boolean), `homeScore`, `awayScore`; `result` is `null` when `isLive` is true (provisional W/L would be misleading). Status filter: `g.status` matches `%final%` OR `%In Progress%` OR `%End of Period%` OR `%Halftime%`. Cached 60s (`top-performances:{league}:{type}:{window}:{sort}:{position}:{limit}`); 30s for `window=today`. Filters preseason/exhibition; tiebreaker `s.playerid ASC`.
- `GET /search`
- `GET /live/:league/games` ‚Äî SSE stream; heartbeat `: ping` every 15s; `event: done` when no live games
- `GET /live/:league/games/:gameId` ‚Äî SSE stream; `event: done` when game is Final
- `GET /games/:id/ai-summary` ‚Äî **requires `Authorization: Bearer <token>` header**; NDJSON stream (`application/x-ndjson`); emits `bullet` (one per insight), `full` (cached ‚Äî full text at once), `done`, or `error` lines; DB-cached after first generation
- `GET /favorites` ‚Äî requires auth; returns `{ players: [...], teams: [...] }`
- `GET /favorites/check?playerIds=1,2&teamIds=3,4` ‚Äî requires auth
- `POST /favorites/players/:playerId` ‚Äî requires auth
- `DELETE /favorites/players/:playerId` ‚Äî requires auth
- `POST /favorites/teams/:teamId` ‚Äî requires auth
- `DELETE /favorites/teams/:teamId` ‚Äî requires auth
- `GET /user/profile` ‚Äî requires auth; returns `id`, `email`, `first_name`, `last_name`, `default_league`
- `PATCH /user/profile` ‚Äî requires auth; body `{ firstName, lastName, defaultLeague }`; validates `defaultLeague` against `["nba", "nfl", "nhl"]`
- `DELETE /user/account` ‚Äî requires auth; deletes Supabase auth user then DB row (cascades favorites)
- `POST /webhooks/supabase-auth` ‚Äî verified by `Authorization: <SUPABASE_WEBHOOK_SECRET>`; inserts user on signup
- `POST /chat` ‚Äî **requires auth**; SSE stream; body `{ message, conversationId?, pageContext? }`; emits `delta`, `status`, `done`, `error` events; rate-limited by `chatLimiter` (30 req/15 min prod)

## Frontend routes
- `/` ‚Üí Homepage
- `/about` ‚Üí About (lazy-loaded)
- `/privacy` ‚Üí PrivacyPage (lazy-loaded)
- `/:league` ‚Üí LeaguePage
- `/:league/teams/:teamId` ‚Üí TeamPage
- `/:league/players/:slug` ‚Üí PlayerPage
- `/:league/games/:gameId` ‚Üí GamePage
- `/auth/callback` ‚Üí AuthCallback (OAuth popup handler ‚Äî no layout shell)
- `/:league/compare` ‚Üí ComparePage (lazy-loaded; query params `type=players|teams` and `ids=slug1,slug2`)
- `/pulse` ‚Üí PulsePage (lazy-loaded; top-level. Sub-tabs: **Highlights** (NBA-only leaderboards with internal Best Games / Last 7 Days mode toggle) and **Reports** (cross-league injuries/moves/birthdays/streaks feed). Top-level league pills (All/NBA/NFL/NHL); the Highlights sub-tab is hidden when the league pill is NFL or NHL. League changes animate the active tab content with a slide-in transition. URL params: `?league=`, `?type=` (Reports tab), `?tab=highlights|reports`, `?mode=games|cumulative` (Highlights tab))
- `/reports` ‚Üí redirects to `/pulse` (back-compat for old links)
- `*` ‚Üí ErrorPage (404 catch-all, lazy-loaded)
